CSC 120 Lab 6

In this lab we will make a Java Application that instantiates several objects from a RoundHead class, and provides buttons to control the RoundHeads. We will:

  • Use Boolean conditions and if statements.
  • Use Buttons and ActionListeners.
  • Call methods of one object from methods of another object.
  • Add data members and methods to an existing class.

Part 1: Project Setup

  1. Create a new NetBeans project called Lab6, and copy the starter source files to it.

    1. File | New Project, etc. Make sure the Project Location is in your CSC120 folder, and the Project Name is Lab6.

    2. Right-click on the following link and save this .zip file in your CSC120 folder:
      Lab6StarterFiles.zip
    3. Use Windows Explorer to Extract All from the .zip file you saved in the previous step, and then copy All Four files that were extracted, and paste them in the src folder of your Lab 6 Netbeans project.

  2. Run the program.

    1. Open the file MUFrame.java and switch to "Source" view; then edit the file so that the comment header at the top of the file has your name in place of Samantha Student's name and uses today's date. 

    2. Switch to "Design" view and right-click on the label in the interface that says "Samantha Student" and edit the text so that your name is used instead of Samantha's. 

    3. Change the title of the JFrame for the program so that your name appears instead of Samantha's. 

    4. Open the file MUPanel.java and edit the comment header at the top of the file so that it has your name in place of Samantha's and uses today’s date.

    5. Run the MUFrame.java application.

      1. You should see one RoundHead character and one button, but at this time, clicking the button doesn't do anything. 

      2. Samantha Student's name should not appear anywhere in the window or in the window frame.

Part 2: Study The Code

  1. Look over the RoundHead class definition.

    1. Open the RoundHead.java file in the NetBeans IDE.  Notice the various data properties and the purpose of each.

    2. The constructor expects parameters to initialize some of the data members, but just sets the size to 30 for every RoundHead that is constructed.

    3. The draw method should seem understandable, although a bit involved.  Notice the use of an if statement.

    4. The moveVertically method, when invoked by some other object, will change the value of down, so that the next time draw is invoked the RoundHead will appear in a different position.

Part 3: Revising Code

  1. Get the Move RoundHeads button to work.

    1. The starter program has been set up with a partially completed moveButtonActionPerformed method for you already.  In "Source" view in the MUFrame class, find the moveButtonActionPerformed method definition.  
      Note that this method doesn't do anything right now other than call the repaint method of the muPanel object.
    2. Inside the moveButtonActionPerformed method, as the first action to be taken in the method, add a call to the moveObjects method of the muPanel object — this method accepts NO parameters inside the parentheses of the method call.  Hint, revisit your Lab 5 project if you don't recall how to do this.
      Note:  you need to keep the repaint method call as the last action in moveButtonActionPerformed.
    3. Run the program, test, and fix as needed.  Each time you click the Move RoundHeads button, the RoundHead should move up until it gets close to the top edge of the viewer, then start moving down. If you keep clicking until it gets close to the bottom edge of the viewer, then it should start moving up again.

  2. Add a working Change Mood button to the Program.

      3. Drag the new button so it is located a little bit below the move button.  Edit the text of the new button so it displays "Change Mood", and change the variable name for the button to changeMoodButton.

      5. Run the program now.  You should now see two buttons in the control panel area on the right, but wait a minute –
        clicking the Change Mood button doesn't do anything, because we need to attach an action to this button.  We will add the actions in the next several steps so that the Change Mood button does something.

    2. Add a method to the MUPanel class that will change the mood of all RoundHead objects in the program:

      1. In the MUPanel.java file, find the moveObjects method — copy this entire method, and paste the lines into the file immediately after the moveObjects method. 

      2. Rename the copied method so that it is named changeObjectMoods, and change the method call in the body of this method so that the changeMood method of the firstOne object is invoked.
    3. Add an actionPerformed method for the Change Mood button:

      1. In "Design" view for the MUFrame.java file, double-click on the changeMoodButton to attach an ActionPerformed method.  Edit the body of the changeMoodButtonActionPerformed method so that it calls the changeObjectMoods method of the muPanel object, and then calls repaint for muPanel.

      2. Run the program now, and if everything is working correctly, the buttons should do what their labels say they do!  Test and fix your program as needed so that each time you click the Change Mood button, the RoundHead changes from happy to sad or vice versa.

    3. There should now be a total of 5 RoundHeads that can move around in the application window, and be changed from happy to sad and sad to happy.

  4. Add the ability to move the RoundHeads horizontally.

    1. Edit the RoundHead class.

      1. Add one additional private data member, which will represent the horizontal step size that a RoundHead should take when the move button is clicked.

      2. Modify the constructor method so that it expects one additional parameter, and uses it to initialize the added private data member.

      3. Add a new method called moveHorizontally. You may use the existing moveVertically method as a guide, but now worry about turning around horizontally to avoid going off the left or right of the viewer window rather than worrying about turning around vertically to avoid going off the top or bottom of the viewer window.

    2. Edit the MUPanel class.

      1. In the constructor method of the MUPanel class, change the instantiations of your RoundHeads, adding the required additional parameter.   5. Make the RoundHeads have different sizes and give them nickNames.

      g.drawString(nickName, ?, ?);
    4. Run, test, fix.

Part 5 Challenge:  Grow and Shrink the RoundHeads

  1. Add a method in the RoundHead class called grow.

    1. This method returns nothing (void), and has no parameters. The body of the method makes the size grow by 50%. 
                  size = size + size/2;
    2. Add a method in the MUPanel class called growTheObjects. This method should call the grow method for each of the RoundHead objects.

    3. In the MUFrame class, add another JButton called growButton.  Using the same style as was used for the other buttons in your application, give the growButton meaninful label text, set its colors, add the actionPerformed method, etc. Follow the pattern that was used for the other buttons.

    4. The actionPerformed method for the growButton should call the growTheObjects method of the muPanel object before repainting the muPanel.

    5. Run, test, fix. Try your new button. It should grow each RoundHead every time you click it.

  2. Add a method in the RoundHead class called shrink

    1. Next, add a button to shrink the RoundHeads by 50% each time a button is clicked.  If an individual RoundHead's size is less than 10, it should not shrink when the button is clicked.

    2. Run, test, fix. Try your new button. It should grow each RoundHead every time you click it.
